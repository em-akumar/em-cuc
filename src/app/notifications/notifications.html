Notification Usage
        <div emc="SliderNotifications" options="vm.Notifications"></div>
       
this.Notifications = {

      contentTmpl: '<span class="alert-text-header">Errors Found - </span><span class="alert-text-body">&nbsp;Error message description.</span>', // notification html template

      closeButton: 'true',  // notification close icon 'true' , 'false'

      sliderType: 'danger', // notification type 'error', 'success', 'warning','disabled'

      sliderAutoClose: 'true', // notification auto close 'true', 'false'

      sliderIconFlag: 'true' // notification image icon 'true', 'false'

      sliderPosition: 'fixed' // Optional Notification sticky 'fixed', by default non-sticky 'relative'
      isComplex: 'true', // Optional Notification details (complex) 'true', 'false'
      complexTmpl: 'Complex details template will come here', //Optional notification details html template
      containerEl: '#error-container' //Container element id, Use only if you want to display notification in container level
    };
Property Keys Controller Properties
contentTmpl Add the ability to override the template used on the component.
closeButton Show/Hide notification close icon
sliderType It can be error (color: red), success (color:green), warning (color: orange), disabled (color: gray)
sliderAutoClose True or False, Auto Close the page or container level notification
sliderIconFlag True or False, Show/Hide Slider icon page or container level notification
sliderPosition Fixed for Sticky and relative for non-sticky
complexTmpl Add complex details template
containerEl Need container ID e.g. #error-container, Required for container level notification
